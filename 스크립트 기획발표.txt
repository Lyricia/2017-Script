발표는 개발에 대한 개략적인 소개, 개발과정, 사용할 API 목록, 기능 상세 내역, 일정 순으로 발표하겠습니다.

먼저 이 프로젝트는 평소에 자주 타는 버스 노선을 쉽게 검색할 수 없을까 라는 생각에서 시작하게 되었습니다. 여기에 검색 키워드를 즐겨찾기 하거나 검색 결과를 메일로 보내는 등 제가 지금까지 버스 검색을 하면서 경험했던 내용을 바탕으로 이용 편의성을 높이고자 하였습니다.

개발 과정은 공공데이터 포털에서 구현에 필요할 것으로 보이는 API를 검색하여 내용을 살펴보고 API 신청 및 KEY 발급을 합니다. KEY 발급이 완료된 API에 대하여 모듈 테스트를 진행하여 실제 데이터의 이동을 확인하고 그 모듈을 기반으로 완성된 프로그램을 제작합니다.

프로그램 제작에 사용할 API 목록은 다음과 같습니다.
버스 노선검색, 위치 정보 검색, 정류소 정보 조회 등의 API를 사용할 예정입니다.

기능 상세 내역으로는
가장 먼저 노선을 검색하고 이를 기반으로 데이터베이스를 구축합니다.
이 구축된 데이터 베이스를 기반으로 API를 활용하여 버스, 정류장등의 검색 기능을 완성합니다.
버스노선이나 정류장에 즐겨찾기를 해두고 해당 오브젝트의 접근을 빨리 할 수 있도록 합니다.
검색한 내역에 대해서 일정 시간을 기점으로, 혹은 새로고침을 누름으로써 내용을 갱신 할 수 있도록 합니다.

개발 일정은
1주차에는 API를 조사하고 레이아웃 설계를 합니다.
2주차에는 상세 기능을 디자인하고
3주차에는 API를 연동하여 실제 기능을 구현합니다.
4주차에는 모듈테스트를 끝내고 인터페이스를 만듭니다.
5주차에는 인터페이스와 기능을 결합시키고
6주차에는 메일과 C++연동을 합니다.
7주차에 최종 테스트후 배포파일 작성합니다.
